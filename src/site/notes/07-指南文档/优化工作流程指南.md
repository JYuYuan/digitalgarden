## 🔧 自动化脚本

### 脚本 1：项目启动助手

创建 `~/.zshrc` 或 `~/.bashrc` 别名：

```bash
# 知识库相关命令
export KB_ROOT="/Users/dompling/WebstormProjects/Work/.obsidian-vault"

# 快速查看项目启动信息
kb-start() {
    local project=$(basename $(pwd))
    local doc=$(find "$KB_ROOT" -name "${project}.md" 2>/dev/null | head -1)
    
    if [ -n "$doc" ]; then
        echo "📚 知识库：$project 启动信息"
        echo ""
        sed -n '/## ⚡ 快速启动/,/^##/p' "$doc" | head -n -1
    else
        echo "❌ 当前项目不在知识库中"
    fi
}

# 快速搜索问题记录
kb-search() {
    if [ -z "$1" ]; then
        echo "用法: kb-search <关键词>"
        return 1
    fi
    
    echo "🔍 搜索知识库：$1"
    echo ""
    grep -i --color=always -n "$1" "$KB_ROOT/问题记录.md"
}

# 快速打开项目文档
kb-open() {
    local project=$(basename $(pwd))
    local doc=$(find "$KB_ROOT" -name "${project}.md" 2>/dev/null | head -1)
    
    if [ -n "$doc" ]; then
        open "$doc"
    else
        echo "❌ 当前项目不在知识库中"
    fi
}

# 快速添加问题记录（简化版）
kb-add() {
    if [ -z "$1" ]; then
        echo "用法: kb-add '问题简述' '解决方案'"
        return 1
    fi
    
    local project=$(basename $(pwd))
    local date=$(date +"%Y-%m-%d")
    local time=$(date +"%H:%M")
    
    cat >> "$KB_ROOT/问题记录.md" <<EOF

## $date - $1

**问题描述：**
$2

**解决方案：**
$3

**相关项目：** $project

**标签：** #$project

记录时间：$date $time

---
EOF
    
    echo "✅ 已添加到问题记录"
}

# 查看知识库统计
kb-stats() {
    echo "📊 知识库统计"
    echo ""
    echo "项目文档："
    find "$KB_ROOT" -name "*.md" -path "*/02-Web端项目/*" | wc -l | xargs echo "  Web 端："
    find "$KB_ROOT" -name "*.md" -path "*/03-小程序项目/*" | wc -l | xargs echo "  小程序："
    find "$KB_ROOT" -name "*.md" -path "*/04-移动端项目/*" | wc -l | xargs echo "  移动端："
    echo ""
    echo "问题记录："
    grep -c "^## " "$KB_ROOT/问题记录.md" | xargs echo "  总计："
}
```

**使用示例：**
```bash
# 显示当前项目启动信息
kb-start

# 搜索问题
kb-search "npm install"

# 打开项目文档
kb-open

# 快速添加问题
kb-add "问题标题" "问题描述" "解决方案"

# 查看统计
kb-stats
```

### 脚本 2：自动归档问题记录

创建 `scripts/archive-problems.sh`：

```bash
#!/bin/bash
# 自动归档问题记录

KB_ROOT="/Users/dompling/WebstormProjects/Work/.obsidian-vault"
PROBLEMS="$KB_ROOT/问题记录.md"
ARCHIVE_DIR="$KB_ROOT/07-问题库"
DATE=$(date +"%Y-%m")

# 创建归档目录
mkdir -p "$ARCHIVE_DIR"

# 计算问题数量
count=$(grep -c "^## " "$PROBLEMS")

if [ "$count" -ge 50 ]; then
    echo "📦 问题记录已达 $count 条，开始归档..."
    
    # 备份到归档
    cp "$PROBLEMS" "$ARCHIVE_DIR/问题记录-$DATE.md"
    
    # 清空主文件（保留标题）
    head -n 10 "$PROBLEMS" > "$PROBLEMS.tmp"
    mv "$PROBLEMS.tmp" "$PROBLEMS"
    
    echo "✅ 已归档到: $ARCHIVE_DIR/问题记录-$DATE.md"
    echo "✅ 问题记录已重置"
else
    echo "📊 当前问题记录数：$count/50"
    echo "未达到归档阈值"
fi
```

**使用：**
```bash
chmod +x scripts/archive-problems.sh
./scripts/archive-problems.sh
```

### 脚本 3：项目文档快速生成

创建 `scripts/quick-doc.sh`：

```bash
#!/bin/bash
# 快速生成项目文档框架

if [ -z "$1" ]; then
    echo "用法: ./quick-doc.sh <项目名>"
    exit 1
fi

PROJECT=$1
KB_ROOT="/Users/dompling/WebstormProjects/Work/.obsidian-vault"
DOC_FILE="$KB_ROOT/02-Web端项目/$PROJECT.md"

# 检查是否已存在
if [ -f "$DOC_FILE" ]; then
    echo "⚠️  文档已存在：$DOC_FILE"
    exit 1
fi

# 生成文档框架
cat > "$DOC_FILE" <<EOF
# $PROJECT

> 项目简介

---

## ⚡ 快速启动

\`\`\`bash
# Node 版本
Node 16.x+

# 安装依赖
npm install

# 启动命令
npm run dev

# 访问地址
http://localhost:3000
\`\`\`

---

## 📋 项目信息

- **项目名称**：$PROJECT
- **项目类型**：Web 前端应用
- **开发状态**：#维护中
- **项目路径**：\`/Users/dompling/WebstormProjects/Work/$PROJECT\`

---

## 🛠️ 技术栈

待补充...

---

## 🔖 标签

#前端 #Web端 #维护中

---

## 相关链接

- [[技术栈索引]]
- [[项目总览]]
- [[项目关系图]]
EOF

echo "✅ 已创建文档：$DOC_FILE"
echo "📝 请在 Obsidian 中打开并补充详细信息"
```

**使用：**
```bash
chmod +x scripts/quick-doc.sh
./scripts/quick-doc.sh "new-project-name"
```

### Raycast Extension

**功能：** 快速搜索知识库

#### 步骤 1：安装 Raycast Script Commands

```bash
# 创建脚本目录
mkdir -p ~/Documents/Raycast/Scripts

# 创建搜索脚本
cat > ~/Documents/Raycast/Scripts/kb-search.sh <<'EOF'
#!/bin/bash

# Required parameters:
# @raycast.schemaVersion 1
# @raycast.title Search Knowledge Base
# @raycast.mode fullOutput

# Optional parameters:
# @raycast.icon 📚
# @raycast.argument1 { "type": "text", "placeholder": "搜索关键词" }

kb_root="/Users/dompling/WebstormProjects/Work/.obsidian-vault"
query="$1"

echo "🔍 搜索知识库：$query"
echo ""

# 搜索项目文档
find "$kb_root" -name "*.md" -type f -exec grep -l "$query" {} \; | while read file; do
    echo "📄 $(basename "$file")"
    grep -n --color=never "$query" "$file" | head -3
    echo ""
done
EOF

chmod +x ~/Documents/Raycast/Scripts/kb-search.sh
```

#### 步骤 2：在 Raycast 中导入

1. 打开 Raycast
2. Extensions → Script Commands
3. 添加脚本目录

#### 使用：
```
Raycast → 输入 "Search Knowledge Base" → 输入关键词
```

## 📊 工作流程对比

### 优化前 vs 优化后

#### 场景：切换项目并启动

**优化前（5-10 分钟）：**
```
1. cd 到项目目录
2. 找 README.md 或 package.json
3. 查看启动命令
4. 检查 Node 版本
5. 安装依赖
6. 启动项目
7. 查找访问地址
```

**优化后（30 秒）：**
```
1. cd 到项目目录
2. 执行 kb-start
3. 复制启动命令
4. 启动项目
```

或者：
```
1. 对 Claude Code 说："启动 andromeda-web"
2. 复制命令
3. 启动项目
```

---

#### 场景：遇到问题

**优化前（10-30 分钟）：**
```
1. Google 搜索
2. 尝试多个解决方案
3. 记不住下次又搜索
```

**优化后（3-5 分钟）：**
```
1. kb-search "关键词" 或问 Claude Code
2. 如果找到 → 直接用
3. 如果没找到 → 解决后用 kb-add 记录
4. 下次直接查知识库
```

---

#### 场景：新项目启动

**优化前（15-30 分钟）：**
```

```
